Θελω να κανω μια εργασία σε Java για την σχολή μου. Η εργασία αφορά  την υλοποίηση ενος συστήματος διαχείρισης εργασιών (Task Menagement System) και ο καθηγητής θέλει  μεσω της εργασίας να εξετασει 3 κυρια σημεία: 
1. Τον αντικειμενοστρεφη προγραμματισμο 
2. Την αποθήκευση δεδομένων με μορφή JSON
3. Την γραφική διεπαφή (GUI) χρησιμοποιώντας το πλαίσιο JavaFX

Θα σου δωσω περισσότερες πληροφορίες για κάθε τμήμα της εργασίας:
1. Η κυρια κλάση θα είναι η κλαση Task. Τα πεδία της είναι 1. Τιτλος 2. Περιγραφή 3. Κατηγορία 4. Προτεραιότητα 5. Προθεσμία Ολοκλήρωσης 6. Κατάσταση (όπου υπάρχουν 5 πιθανες καταστάσεις: Open, In Progress, Postponed, Completed, Delayed). Οι μέθοδοι της κλάσης είναι: 1. Δημιουργία 2. Τροποποίηση Στοιχείων 3. Διαγραφή.
Οπως ανέφερα, ενα task ανηκει σε μια κατηγορία. Θελουμε ο χρήστης να ορίζει νέες κατηγορίες, να τροποποιήσει και να διαγράφει ήδη υπάρχουσες. Πρεπει να ορίσουμε νέα κλάση για την κατηγορία; Αν ναι με τι κληρονομικότητα?
Επίσης, καθε task εχει προτεραιότητα. Θέλουμε ο χρήστης να μπορεί να ορίζει νέα επίπεδα προτεραιότητας δίνοντας το σχετικό όνομα (το αρχικο ειναι default). Επίσης θελουμε να μπορεί να τροποποιήσει ενα ονομα 
ενος επιπέδου προτεραιότητας ή και να διαγράψει ενα. Πρεπει να δημιουργήσουμε νέα κλάση για τα επίπεδα προτεραιότητας?
Επιπλέον το σύστημα μας υποστηρίζει υπενθυμίσεις. Ο χρήστης δημιοιυργεί υπενθυμίσεις για τις εργασίες. Μια υπενθύμιση σχετίζεται πάντα με μια εργασία. Υπάρχουν 4 πιθανοί τυποι υπενθυμίσεων οι οποίοι σχετίζονται με την προθεσμία. Σιγουρα λοιπόν πρεπει να υπάρχει μια νέα κλάση για τις υπενθυμίσεις. Ο χρηστης πρεπει επίσης να μπορεί να τροποποιήσει ή και να διαγράψει μια υπενθύμιση
Τέλος ο χρήστης πρεπει να έχει την δυνατότητα αναζήτησης εργασιων με βαση συνδυασμό των κριτηρίων τιτλος / κατηγορία / προτεραιότητα  

2. Σχεττικα με την αποθηκευση και ανακτηση των δεδομένων θα χρησιμοποιήσω JSON. Αρχικά πρεπει να ορίσω την δική μου οργάνωση (data schema) για τα JSON δεδομένα καθώς και το σύνολο των αρχειων που θα χρησιμοποιηθούν για την αποθήκευση των δεδομένων. Τα αρχέια με τα δεδομένα θα αποθηκεύονται σε εναν φάκελο με όνομα "medialab". Επειτα θελω τις μεθόδους (μέσω των κατάλληλων κλάσεων) που να μου επιτρέπουν αυτή την ανακτηση της πληροφορίας. Επίσης χρειάζομαι να διατηρείται η συνολική κατάσταση των δεδομένων μεταξύ διαδοχικών εκτελέσεων της εφαρμογής. Η υλοποίηση επομένως στηρίζεται σε 3 στάδια: Αρχικοποίηση εφαρμογης (ανάκτηση της πληροφορίας απο τα json και αντίστοιχων αρχικοποίηση αντικειμένων )/ Εκτέλεση εφαρμογής (Η εφαρμογή θα χρησιμοποιεί τις πληροφορίες κατάστασης
που έχουν ανακτηθεί στην μνήμη του προγράμματος κατά την αρχικοποίση )/ Τερματισμός εφαρμογής (όπου γίνεται η ανανέωση των json αρχείων )

3. To GUI θα έχει περίπου αυτή την μορφή: 1 Κεντρικό παράθυρο χωρισμένο σε 2 βασικά μερη. Στο πανω μερος εμφανίζονται 4 βασικές πληροφορίες με βαση τις ενέργειες του χρήστη. Στο άλλο μέρος εμφανίζονται οι λειτυργίες της γραφικής διεπαφής: Διαχείριση Εργασιών / Διαχείριση κατηγοριών / Διαχείριση επιπέδων προτεραιότητας / Διαχείριση υπενθυμίσεων / Αναζήτηση εργασιών
